--- layout: post title: git 'er done! categories: [] tags: [] status: private type: post published: false meta: _edit_last: '38' --- If you've been in development for any significant length of time, you've run into source control systems, like Starteam, Visual Source Safe, CVS, or Subversion (SVN). Maybe you've even used one or two. There's a new trend in source control called "distributed source control". Traditionally, you have a single central server that houses all your code, manages synchronization, and the like. However, new source control systems like git are taking a different approach - there is no single repository. Instead, every developer in the project runs their own repository, which they may allow others to pull code from. It's a little confusing at first, but far more flexible once you get into it. You've may have heard of git, but how do you practically use it? It's easy! Once installed, you have a few basic commands you might be interested in.

git init

`git init` creates a new repository in your current working directory. For example: Let's say I have an empty directory, and want to create a git repository out of it. I just `git init` and we're ready to start committing code.

git add

Now that we have a repository, how do we use it? It's easy. Let's add a file to the directory - call it README - and put "Hello, Version 1" in it. Save it, and next, we'll add it to our git repository. 
`git add README`

git clone

`git clone` allows you to clone another existing repository. Let's say you want to get a copy of my site_config Rails plugin. All you'd have to do is clone the URL that github gives you: 
`git clone git://github.com/cheald/site_config.git`
Once the command completes, you'll have a brand-new `site_config` directory, with the project files inside of it.
